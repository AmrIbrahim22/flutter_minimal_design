# Contributing to Flutter Minimal Design

We love your input! We want to make contributing to Flutter Minimal Design as easy and transparent as possible.

## Development Process

1. Fork the repo
2. Create your feature branch (`git checkout -b feature/AmazingFeature`)
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`)
4. Push to the branch (`git push origin feature/AmazingFeature`)
5. Open a Pull Request

## Any contributions you make will be under the MIT Software License

When you submit code changes, your submissions are understood to be under the same [MIT License](http://choosealicense.com/licenses/mit/) that covers the project.

## Report bugs using Github's [issue tracker]

We use GitHub issues to track public bugs. Report a bug by [opening a new issue](https://github.com/yourusername/flutter_minimal_design/issues).
